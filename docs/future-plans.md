## Phase 4: 表示時のバリエーション

### コンセプト

同じ文字でも毎回違う字形で表示される「手書きらしさ」の再現。

**現在のcalt機能の限界**:
- 文脈依存で切り替わるが「ランダム」ではない
- 「ああああ」は常に同じパターン

**アプリ側での動的切り替え**:
- フォントファイルには複数グリフを格納
- 表示時にJavaScriptでランダム選択
- 同じ文章でも表示するたびに違う見た目

**実装案**:

```typescript
// Web版
function renderWithVariation(text: string, font: Font) {
  return text.split('').map(char => {
    const variants = font.getVariants(char);
    const selected = variants[Math.floor(Math.random() * variants.length)];
    return `<span style="font-feature-settings: 'salt' ${selected.index}">${char}</span>`;
  }).join('');
}
```

**対応プラットフォーム**:
- Web: CSS + JavaScript
- デスクトップ: 専用ビューアアプリ
- モバイル: PWA or ネイティブアプリ

## Phase 5: 高度な機能

### クラウド保存（オプトイン）

**機能**:
- ユーザーが希望すれば途中経過を保存
- 複数デバイスでの作業継続
- 過去に作成したフォントの管理

**プライバシー**:
- デフォルトはオフ（ローカルのみ）
- ユーザーが明示的に同意した場合のみ有効
- 暗号化保存

### フォントの微調整エディタ

**機能**:
- 生成後にグリフを個別調整
- カーニング設定（文字間隔）
- ベースライン調整
- サイズ調整
