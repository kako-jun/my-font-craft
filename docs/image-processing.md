# 画像処理仕様

## 入力要件

| 項目               | 要件                                                 |
| ------------------ | ---------------------------------------------------- |
| 対応形式           | JPEG, PNG, WebP（HEICはSafariのみ対応）              |
| 推奨解像度         | 300dpi以上                                           |
| 最小解像度         | 150dpi（警告表示）                                   |
| 最大ファイルサイズ | 50MB/枚                                              |
| 入力方法           | フォルダ選択 or ZIPファイル（ドラッグ&ドロップ対応） |
| ファイル名規則     | 任意（QRコードでページ識別。フォルダ階層も自由）     |

**ユーザーによる事前補正は不要**。スキャナーでもスマホ撮影でも対応。
ZIP内の `__MACOSX/` や `._*` メタファイルは自動除外。

---

## 処理フロー

```
1. 画像読み込み
      ↓
2. 四隅マーカー検出 → 向き検出・回転補正 → 台形補正
      ↓
3. QRコード読み取り → ページ識別（補正後の画像から読む）
      ↓ （読めない場合はエラー）
4. グレースケールバー読み取り → 色補正閾値決定
      ↓
5. シアンサンプル読み取り → 除去閾値決定
      ↓
6. シアン色の除去
      ↓
7. 各マス切り出し（48文字 × 2マス = 96マス）
      ↓
8. 各マスごとに:
   - 空マス検出 → スキップ
   - チェック欄解析（✓/×/空欄）
   - 採用判定
      ↓
9. 採用マスの文字を二値化
      ↓
10. 輪郭抽出 → ベクター化
      ↓
11. グリフとして登録
```

---

## 台形補正

### 四隅マーカー検出

1. 画像を二値化
2. 輪郭検出で四隅の★☆マーカーを探索
3. 4点の座標を取得
4. 射影変換（Perspective Transform）で長方形に補正

### 向き検出

- 左上マーカーのみ塗りつぶし（★）
- 他は白抜き（☆）
- 塗りつぶしの位置で向きを判定（0°/90°/180°/270°）

---

## 色補正

### グレースケールバーによる補正（将来実装）

**NOTE**: Phase 1 ではグレースケールバーの色補正は未実装。テンプレートにバーは印刷されるが、スキャン時の補正には使用していない。

1. グレースケールバーの10段階を読み取り
2. 各段階の実際のRGB値を取得
3. 期待値とのマッピングテーブルを作成
4. 画像全体に適用

```
例：
期待: 100% = #000000, 50% = #808080
実際: 100% = #1A1A1A, 50% = #909090
→ 全体を暗く補正
```

### シアン除去

1. シアンサンプルのRGB値を取得
2. 許容範囲を設定（色距離 ≤ 80）
3. 許容範囲内のピクセルを白(#FFFFFF)に置換

## マス切り出し

### 座標計算

テンプレート仕様に基づき、各マスの座標を計算：

```typescript
const MARGIN = 10; // mm
const HEADER_HEIGHT = 15; // mm
const MARKER_MARGIN = 5; // mm
const COL_WIDTH = 60; // mm
const ROW_HEIGHT = 25; // mm
const SAMPLE_WIDTH = 10; // mm
const CELL_SIZE = 20; // mm
const CELL_GAP = 2; // mm

function getCellPosition(row: number, col: number, cellIndex: number) {
  const x = MARGIN + col * COL_WIDTH + SAMPLE_WIDTH + CELL_GAP + cellIndex * (CELL_SIZE + CELL_GAP);
  const y = MARGIN + HEADER_HEIGHT + MARKER_MARGIN + row * ROW_HEIGHT;
  return { x, y };
}
```

## チェック欄解析

### AI使用方針

| 用途             | AI使用      | 理由                     |
| ---------------- | ----------- | ------------------------ |
| 文字の補間・生成 | ❌ 使わない | 本人の字を100%使用が必須 |
| ✓印の検出        | ✅ 使う     | 手書きの揺らぎに対応     |
| ×印の検出        | ✅ 使う     | 手書きの揺らぎに対応     |
| 空マスの検出     | ✅ 使う     | ノイズと記入の区別       |

### 実装方針（Phase別）

**Phase 1（MVP）**: テンプレートマッチング

- 開発コストが低い
- 精度は十分
- ブラウザで動作可能

**Phase 2（将来）**: 画像分類（機械学習）

- より高精度
- TensorFlow.jsで実装可能

### 空マス検出の閾値

- 二値化後、黒ピクセルが全体の**1%未満** → 空マスと判定
- ノイズ除去: 連続した黒領域が**10ピクセル未満**は無視

### ✓/×の検出手順

1. チェック欄（3mm高さ）を切り出し
2. 二値化
3. テンプレートマッチングで判定:
   - ✓テンプレ���トと��類似度 > 閾値 → ✓と判定
   - ×テンプレートとの類似度 > 閾値 → ×と判定
   - どちらも低い → 空欄と判定

## ベクター化

### アルゴリズム

1. **二値化**（大津の方法）
2. **輪郭抽出**（OpenCVのfindContours相当）
3. **輪郭の簡略化**（Douglas-Peuckerアルゴリズム、反復版。epsilon は画像サイズに応じて動的スケーリング: `max(1.0, min(width, height) / 80)`）
4. **ベジェ曲線への変換**

### 正規化

| 項目         | 値        |
| ------------ | --------- |
| Units per Em | 1000      |
| グリフの高さ | 800 units |
| ベースライン | 0         |

文字の実際のバウンディングボックスを800 unitsにスケーリング。

## エラーハンドリング

### エラー種別と対処

| エラー                         | 対処                               |
| ------------------------------ | ---------------------------------- |
| QRコードが読めない             | エラー表示、該当画像をスキップ     |
| マーカーが検出できない         | 警告表示、手動補正を促す           |
| 解像度が低い                   | 警告表示、処理は継続               |
| 文字が全く書かれていないページ | スキップ（警告なし）               |
| 不正な画像形式                 | エラー表示、該当ファイルをスキップ |

### エラーメッセージ一覧

| エラー             | メッセージ                                                                                                | リカバリー手順                                       |
| ------------------ | --------------------------------------------------------------------------------------------------------- | ---------------------------------------------------- |
| QRコード読取失敗   | 「画像 X のQRコードを読み取れませんでした。画像が不鮮明な可能性があります。」                             | 該当ページを再スキャン、または手動でページ番号を入力 |
| マーカー検出失敗   | 「画像 X の四隅マーカーが見つかりません。用紙全体が写っているか確認してください。」                       | 手動で四隅をクリックして指定                         |
| 低解像度           | 「ページ X の解像度が低いです（推奨: 300dpi以上）。処理は継続しますが、品質が低下する可能性があります。」 | 再スキャン推奨                                       |
| 不正ファイル形式   | 「ファイル X は対応していない形式です。JPEG, PNG, WebP, HEIC形式を使用してください。」                    | 正しい形式に変換                                     |
| ファイルサイズ超過 | 「ファイル X が大きすぎます（最大: 50MB）。」                                                             | 画像を圧縮                                           |

## パフォーマンス目標

| 処理                     | 目標時間 | 根拠                               |
| ------------------------ | -------- | ---------------------------------- |
| 画像処理（1ページ）      | 2秒以内  | ユーザーが進捗を体感できる最小単位 |
| 全ページ処理（51ページ） | 2分以内  | Webアプリの許容待機時間（5分未満） |
| フォント生成             | 10秒以内 | 処理完了の即時フィードバック       |

### 根拠

- **2秒/ページ**: プログレスバーが定期的に動くことでユーザーの離脱を防止
- **3分以内**: 研究によるとWebアプリの待機許容時間は3〜5分程度
- **10秒以内**: 最終処理は短くして達成感を演出

※ 実際のベンチマークで調整予定

## ユーザーフロー（3フェーズ）

```
Phase 1: スキャン
  画像アップロード → マーカー/QR/シアン処理 → セル切出し → ベクター化
  → 結果をグリッドに表示（ページ単位→文字単位の2階層）

Phase 2: 確認（レビュー）
  グリッド上で取得/未検出/インポート済みを確認
  ├── 全文字取得 → Phase 3 へ
  ├── 未検出あり → リトライ用テンプレートPDFダウンロード
  │    → 書き直し → 追加スキャン → 既存結果にマージ → 再確認
  ├── 既存フォントをインポート → 未検出分をインポートで穴埋め
  └── そのまま生成 → Phase 3 へ（未検出文字はフォントに含まれない）

Phase 3: フォント生成
  ユーザーがボタンを押して明示的に実行 → .ttf ダウンロード
```

### リトライ用テンプレートPDF

未検出文字のみを集めた小規模テンプレート（例: 6文字なら1ページ）を生成。
51ページ全体の再印刷を不要にする。

### 追加スキャンのマージ

レビュー中に追加画像をアップロードすると、既存結果を保持したまま
新しく取得できた文字で未検出分を差分マージする。
